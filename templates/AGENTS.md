## Workflow

- Read `PRODUCT.md` and `DESIGN.md` before substantial frontend work.
- Plan before 3+ step tasks.
- Use focused tests for touched behavior.
- Run typecheck and lint before done where practical.
- Do not revert user changes unless explicitly requested.

## Frontend Standards

- Use existing layouts, tokens, and primitives first.
- Keep API calls in hooks or clients.
- Cover loading, empty, error, permission, submitting, and success states.
- Avoid raw colors in product screens.
- Avoid nested cards and decorative dashboard filler.
