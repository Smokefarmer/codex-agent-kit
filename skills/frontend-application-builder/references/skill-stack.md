# Skill Stack

Use the smallest stack that answers the request.

## Default

- `frontend-application-builder`: default for real app workflow screens.
- Project overlay skill: use when the repo has one, for domain rules and product-specific design language.

## Optional Design Lenses

- Impeccable: use for audit, polish, distill, harden, and anti-pattern critique.
- Taste/design-direction skills: use when a screen feels generic or needs stronger visual judgment.
- Animation/design-engineering skills: use for interaction, transitions, and motion performance.

## Component Discovery

- Use shadcn MCP or CLI for primitive discovery and installation.
- Adapt installed primitives to the local token system, copy, permissions, API contracts, and tests.
- Do not let a registry component define the product language by itself.

## Public vs Private

Public skills should contain reusable process, design heuristics, and setup instructions.
Private repo skills should contain business logic, customer language, domain rules, screenshots, schemas, and internal workflow constraints.
